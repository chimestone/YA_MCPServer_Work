### 项目结构

- `core`: 核心业务逻辑模块，包含数据预处理、模型选择、评估的核心算法实现，提供统一的调用接口
- `tools`: 实现上述Tool列表中的三个工具，封装为可被MCP客户端调用的函数
- `config.yaml`: 项目配置文件，除基础元数据外，添加了模型权重路径、评估指标阈值、数据存储目录等额外配置
- `models/`: 新增目录，用于存放预训练模型权重与自定义模型实现
- `data/`: 新增目录，用于存放示例数据集与用户上传数据的临时存储
- `docs/`: 新增目录，用于存放API文档与用户使用手册

### 其他需要说明的情况

- 在 `sops` 模块中添加的密钥变量分别用于：
- `MODEL_API_KEY`：调用外部模型服务（如OpenAI API）的身份验证
- `DATA_STORAGE_KEY`：加密存储敏感数据集与模型权重
- 是否使用了 PyTorch、Tensorflow 等深度学习框架：**未使用**，本项目聚焦于传统机器学习分类算法，便于快速部署与理解
- 是否使用了机器学习、深度学习模型：**使用了机器学习模型**，包括逻辑回归、SVM、随机森林、XGBoost、LightGBM等经典分类算法，未使用深度学习模型
